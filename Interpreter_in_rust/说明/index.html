解释器​

有的解释器很精简，没有解析步骤，只有单纯的解释输入。

有的解释器设计的很精妙，如有些解释器进行了高层次优化，使用了高级解析和求值技术；有些解释器不会直接求值，而是将输入编译成称为字节码的内部形式，之后再求值，有更高级的JIT解释器，会将需要的输入即时转换成机器码。

解释器种类繁多并且各不相同很难给出一个明确的定义，但所有解释器都至少有一个相同的功能：输入文本得到一些输出，将源代码作为输入并求值，中途不会产生任何可见且之后还能再次运行的中间结果。

书中构建的解释器为树遍历（tree-walking）解释器，这类解释器会解析源代码，构建抽象语法树（AST），然后对这棵树进行求值。之所以叫做树遍历解释器，是因为其工作方式树在AST上遍历并解释的。

解释器主要包含以下几主要部分：

  • 词法分析器
  • 语法分析器
  • 抽象语法树（AST）
  • 内部对象系统
  • 求值器

书中主要讲解了如何构建自己的词法解析器、语法分析器、树表示形式和求职器（evaluator），其中涉及词法单元（token）、抽象语法树、如何构建抽象语法树、如何求值，以及如何使用新的数据结构和内置函数扩展所用的编程语言。

Monkey 语言​

每个解释器都用来解释一种特定的语言，这也是实现一种编程语言的方式，没有对应的编译器或者解释器，一门编程语言只是一个想法或者一种规范。

书中用到的语言叫做Monkey, 书中为实现解释器专门设计的语言，唯一的实现也是书中的解释器。

Monkey具有以下特性：

  • 类C语法

  • 变量绑定

  • 整型和布尔型

  • 算术表达式

  • 内置函数

  • 头等函数和高阶函数

  • 闭包

  • 字符串数据结构

  • 数组数据结构

  • 哈希数据结构

基本语法如下：

变量声明与值绑定

let age = 1;
let name = "Monkey";
let result = 10 * (20 / 2);

// 数组
let myArray = [1, 2, 3, 4, 5];
// 哈希
let thorsten = {"name": "Thorsten", "age": 28};
// 通过索引表达式访问数组和哈希表
myArray[0]       // => 1
thorsten["name"] // => "Thorsten”

匿名函数

let add = fn(a, b) { return a + b; };

// 省略return, 隐式返回值
let add = fn(a, b) { a + b; };

// 函数调用
add(1,2);

递归

let fibonacci = fn(x) {
  if (x == 0) {
    0
  } else {
    if (x == 1) {
      1
    } else {
      // 使用了fibonacci自身
      fibonacci(x - 1) + fibonacci(x - 2);
    }
  }
};

把函数当作其他函数的参数

let twice = fn(f, x) {
  return f(f(x));
};

let addTwo = fn(x) {
  return x + 2;
};

// 函数调用
twice(addTwo, 2); // => 6”
