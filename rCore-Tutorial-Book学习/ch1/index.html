报错原因：目标平台上没有Rust标准库std, 也不存在任何OS支持的系统调用。

我们将这样的平台称之为裸机平台（bare-metal） 好在是，Rust除了std库之外，还有一个不需要任何OS支持的核心库Core， 它提供了基本类型、数据结构（切片等）、函数、异常处理、unsafe代码支持、裸指针、内存管理、原子操作、所有权、生命周期等，主要用于编写操作系统、嵌入式系统和其他不依赖与标准库的低级代码。

移除std和main​


// 移除标准库，即告诉Rust编译器不在使用标准库，只使用核心库Core
#![no_std]

// 程序启动时，编译器将会查找start lang_item，它定义了程序的入口点(entry pointer)
// start语义是在std库中定义的
// 这行标注是告诉编译器不再使用标准库的start语义
#![no_main]

use core::panic::PanicInfo;

// 返回值为!的是一个发散函数，没有任何返回值，函数永远不会将控制内容返回给调用者
// 这个类型无法被失例化，因为此类型可能具有所有可能值的集合空，可以被转化为任何类型
// 如永远循环（如 loop {}）的函数（如网络服务器）或终止进程的函数（如 exit()）的返回类型都是发散函数
// panic_handler定一个panic处理函数，当发生panic时将会调用该方法
#[panic_handler]
fn panic(_info: &PanicInfo) -> ! {
    // 目前先不进行任何处理
    loop {}
}

// 移除了标准库的start语义，就可以将main函数删除
// fn main() {
//     // println! 宏是有std库提供的，先注释掉
//     // println!("Hello, world!");
// }

对二进制进行反汇编

结果为空，说明这个二进制虽然是合法的，但是其实是一个空程序，其中没有任何内容，原因是薛绍了编译器规定的入口函数_start

构建用户态执行环境（user mode）​

添加_start函数​

// no_mangle: 禁用名称重整，这是为了确保Rust编译器输出的是_start函数，否则对函数名进行重命名，
//    如_ZN3blog_os4_start7hb173fedf945531caE, 这会使得链接器无法辨认
// 编写程序入口
#[no_mangle]
extern "C" fn _start() {
    loop {}
}

重新编译并进行反汇编

$ cargo build
   Compiling os v0.1.0 (/Users/mqy/Workspace/programming/programming_study/os_rcore/my_os/rv-kernel/os)

反汇编出的两条指令就是一个死循环，这已经说明编译器生成的程序已经是一个合法程序，可以使用qemu-riscv64来执行这个程序，如：

$ qemu-riscv64 target/riscv64gc-unknown-none-elf/debug/os
