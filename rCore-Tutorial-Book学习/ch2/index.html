SBI是RISC-V的一种底层规范，操作系统内核与RustSBI的关系和应用程序与操作系统内核的关系类似，SBI为操作系统内核提供服务，只是SBI提供的服务较少，如关机、显示字符串等。

应用程序访问操作系统提供的调用指定是ecall，操作系统访问RustSBI提供的SBI调用的指令也是ecall虽然指令一样，但是所处的特权级不太一样，用用程序位于最弱的用户特权级 User Mode，操作系统位于内核特权集 Supervisor Mode，RustSBI位于及其特权集 Machine Mode

编译执行程序​

# 编译生成ELF格式的执行文件
cargo build --release
# 把ELF执行文件转成bianary文件
rust-objcopy --binary-architecture=riscv64 target/riscv64gc-unknown-none-elf/release/os --strip-all -O binary target/riscv64gc-unknown-none-elf/release/os.bin
# -bios $(BOOTLOADER) 意味着硬件加载了一个 BootLoader 程序，即 RustSBI
# -device loader,file=$(KERNEL_BIN),addr=$(KERNEL_ENTRY_PA) 表示硬件内存中的特定位置
#   $(KERNEL_ENTRY_PA) 放置了操作系统的二进制代码 $(KERNEL_BIN) ，即二进制代码在内存中的位置, $(KERNEL_ENTRY_PA) 的值是 0x80200000
qemu-system-riscv64 \
-machine virt \
-nographic \
-bios ../bootloader/rustsbi-qemu.bin \
-device loader,file=target/riscv64gc-unknown-none-elf/release/os.bin,addr=0x80200000

  1. CPU 的其它通用寄存器清零，而 PC 会指向 0x1000 的位置，这里有固化在硬件中的一小段引导代码， 它会很快跳转到 0x80000000 的 RustSBI 处

  2. RustSBI完成硬件初始化后，会跳转到 $(KERNEL_BIN) 所在内存位置 0x80200000 处， 执行操作系统的第一条指令。

链接器和linker脚本​

链接器（linker）是一个程序，它将生成的目标文件组合为一个可执行文件。不同的操作系统如 Windows、macOS、Linux，规定了不同的可执行文件格式。

数据段：主要由三部分组成（.bass段、.data段、.rodata段），.bass主要存放未初始化静态数据；.data存放已经初始化静态数据；.rodata存放只读数据（常量）。

代码段：主要分两部分（.text和.init），.text存放用户程序代码；.init存放可执行程序初始化代码。
