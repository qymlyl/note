dashboard安装​

对应k8s版本：v1.21.4+k3s1

GitHub: https://github.com/kubernetes/dashboard/tree/v2.3.1

下载部署文件

$ wget https://raw.githubusercontent.com/kubernetes/dashboard/v2.3.1/aio/deploy/recommended.yaml

安装

$ kubectl apply -f recommended.yaml

查看dashboard状态

$ kubectl get pods -n kubernetes-dashboard
NAME                                         READY   STATUS    RESTARTS   AGE
dashboard-metrics-scraper-856586f554-2vb4m   1/1     Running   0          28s
kubernetes-dashboard-865bddc59c-59q5f        1/1     Running   0          28s

查看svc

kubectl get svc -n kubernetes-dashboard
NAME                        TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)    AGE
kubernetes-dashboard        ClusterIP   10.43.181.192   <none>        443/TCP    71s
dashboard-metrics-scraper   ClusterIP   10.43.81.128    <none>        8000/TCP   71s

暴露端口，将type: ClusterIP修改为type: NodePort

$ kubectl -n kubernetes-dashboard edit service kubernetes-dashboard

再次查看svc

$ kubectl get svc -n kubernetes-dashboard
NAME                        TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)         AGE
dashboard-metrics-scraper   ClusterIP   10.43.81.128    <none>        8000/TCP        3m38s
kubernetes-dashboard        NodePort    10.43.181.192   <none>        443:30443/TCP   3m38s

访问 dashboard​

想要访问dashboard服务，需要有访问权限，创建kubernetes-dashboard管理员角色

$ vim dashboard-svc-account.yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: dashboard-admin
  namespace: kube-system
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: dashboard-admin
subjects:
  - kind: ServiceAccount
    name: dashboard-admin
    namespace: kube-system
roleRef:
  kind: ClusterRole
  name: cluster-admin
  apiGroup: rbac.authorization.k8s.io

或者匿名用户

apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: dashboard-anonymous-admin
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: User
  name: system:anonymous
  apiGroup: rbac.authorization.k8s.io

执行文件

$ kubectl apply -f dashboard-svc-account.yaml
serviceaccount/dashboard-admin created
clusterrolebinding.rbac.authorization.k8s.io/dashboard-admin created
