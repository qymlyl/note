安装docker镜像仓库​

cd docker-registry

执行脚本

注：需修改setup_registry.sh中的数据存放位置

./setup_registry.sh

配置镜像仓库

# 设置docker私有镜像仓库的域名信息
vim /etc/hosts        
192.168.16.147 docker.oa.com
vim /etc/docker/daemon.json 
添加 
{
 "insecure-registries": ["docker.oa.com:5000"]
}
systemctl daemon-reload && systemctl restart docker

安装k3s​

https://github.com/k3s-io/k3s/tree/v1.21.4%2Bk3s1

安装​

# 默认
curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION=v1.21.4+k3s1 sh -


curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION=v1.21.4+k3s1 INSTALL_K3S_EXEC="server --docker --disable traefik --write-kubeconfig-mode 644" sh -

# 国内镜像
curl -sfL https://rancher-mirror.rancher.cn/k3s/k3s-install.sh | INSTALL_K3S_VERSION=v1.21.4+k3s1 INSTALL_K3S_EXEC="server --docker --disable traefik --write-kubeconfig-mode 644"   INSTALL_K3S_MIRROR=cn sh -

获取token​

cat /var/lib/rancher/k3s/server/node-token

work加入集群​

curl -sfL http://rancher-mirror.cnrancher.com/k3s/k3s-install.sh | INSTALL_K3S_VERSION=v1.18.8+k3s1  INSTALL_K3S_EXEC="agent --flannel-iface wlan0 --docker --disable traefik" INSTALL_K3S_MIRROR=cn  K3S_URL=https://10.0.0.18:6443 K3S_TOKEN=K10d647b2b9964d759ffb54581236d57af7d18fd798cd87f3c7d9e6f2dd861c74b9::server:05112474d0b177915915752629e8f1a3 sh

检查​

执行如下命令：
kubectl get node 
kubectl get pod -A

卸载​

卸载 K3s Server

sudo /usr/local/bin/k3s-uninstall.sh

卸载 K3s Agent，如果你正在运行 K3s agent 节点，可以使用以下命令来卸载 K3s agent：

/usr/local/bin/k3s-agent-uninstall.sh

卸载残留文件

sudo rm -rf /etc/rancher/k3s /var/lib/rancher/k3s /var/log/k3s.log

检查进程

ps aux | grep k3s

dashboard安装​

对应k8s版本：v1.21.4+k3s1

GitHub: https://github.com/kubernetes/dashboard/tree/v2.3.1

下载部署文件(部署文件会放在附录)

# 默认的
$ wget https://raw.githubusercontent.com/kubernetes/dashboard/v2.3.1/aio/deploy/recommended.yaml

关闭token登录​

修改recommended.yaml，找到kubernetes-dashboard对应的kind: Deploment， 修改spec.template.spec.containers.args

args:
  # 关闭token登录
  #- --auto-generate-certificates
  - --namespace=kubernetes-dashboard
  #- --enable-skip-login
  #- --disable-settings-authorizer
  - --kubeconfig=/etc/kubernetes/kubeconfig
  # Uncomment the following line to manually specify Kubernetes API server Host
  # If not specified, Dashboard will attempt to auto discover the API server and connect
  # to it. Uncomment only if the default does not work.
  #- --apiserver-host=https://192.168.100.1:6443

修改spec.template.spec.containers.env

 env:
  - name: KUBECONFIG
    value: /etc/kubernetes/kubeconfig

修改spec.template.spec.volumes, 添加

volumes:
  - name: kubeconfig
    hostPath:
        path: /etc/rancher/k3s/k3s.yaml
        type: File

http访问参考文章https://www.cnblogs.com/chen2ha/p/17086962.html

固定端口示例​

nodePort为宿主机上的端口

apiVersion: v1
kind: Service
metadata:
  labels:
    k8s-app: kubernetes-dashboard
  name: kubernetes-dashboard
  namespace: kubernetes-dashboard
spec:
  ports:
  - name: https
    nodePort: 30110
    port: 443
    protocol: TCP
    targetPort: 8443
  - name: http
    nodePort: 31213
    port: 80
    protocol: TCP
    targetPort: 9090
  selector:
    k8s-app: kubernetes-dashboard
  type: NodePort

token获取(方式访问需要)​

想要访问dashboard服务，需要有访问权限，创建kubernetes-dashboard管理员角色

$ vim dashboard-svc-account.yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: dashboard-admin
  namespace: kube-system
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: dashboard-admin
subjects:
  - kind: ServiceAccount
    name: dashboard-admin
    namespace: kube-system
roleRef:
  kind: ClusterRole
  name: cluster-admin
  apiGroup: rbac.authorization.k8s.io

或者匿名用户

apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: dashboard-anonymous-admin
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: User
  name: system:anonymous
  apiGroup: rbac.authorization.k8s.io

执行文件

$ kubectl apply -f dashboard-svc-account.yaml
serviceaccount/dashboard-admin created
clusterrolebinding.rbac.authorization.k8s.io/dashboard-admin created
