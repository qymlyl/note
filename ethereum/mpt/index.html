MPT

MPT概览​

目标​

  • 存储State中的Key-Value对（Key为用户地址，Value为{nonce,balance,storageRoot,codeHash}，用处类似于Patricia Trie）
  • 为State生成一个根哈希，存储在区块头中（用处类似于Merkle Tree）

MPT结构​

  1. key和value都是[]byte类型
  2. Key对应的Value一定是在树的叶子节点存放的。
  3. MPT中root到节点的Path和节点中记录的Key一起构成节点完整的Key(Key = Path+NodeKey)
  4. 分支节点的插槽Index就是Path的一部分

节点类型​

以太坊的Merkle Patricia Tree (MPT) 存在五种节点类型：扩展节点、分支节点、叶子节点、空节点

  1. 扩展节点（Extension Node）：
    • 扩展节点用于表示共享前缀的节点。它由一个16进制数字和一个子节点的哈希值组成。
    • 例如，如果有多个节点都有相同的前缀，那么这些节点的前缀可以被合并到一个扩展节点中。
    • 这样可以节省存储空间，并减少树的深度，提高数据存取的效率。
  2. 分支节点（Branch Node）：
    • 分支节点包含了16个子节点，分别对应16进制数字x00～0xf0。
    • 每个子节点可以是空或者另一个节点的哈希值（用于根据哈希值从Db中获取子节点，子节点可能是扩展节点、分支节点、叶子节点）。
    • 分支节点用于将多个子节点连接起来，形成一个完整的路径，表示数据的存储位置。
    • 分支节点可以看作是[ [Ia, Ib, … I*], value], 其中 [Ia, Ib, ... I*] 称为结点的 索引数组 ，每个元素I*指向对应的子结点。 value 则代表从根节点到当前结点的路径组成的key所对应的值。如果不存在这样一个key，则value的值为空。
  3. 叶子节点（Leaf Node）：
    • 叶子节点存储了键值对数据，其中键通常为32字节的哈希值，值可以是任意长度的数据。
    • 叶子节点包含了实际的数据内容，例如账户的状态信息、智能合约的代码或者其他类型的数据。
  4. 空节点（Null Node）：
    • 空节点表示空值或者空路径。
    • 在MPT中，当某个路径上的节点不存在时，会使用空节点进行占位，以保持树的结构完整性。

示例​

示例如下，存储指定的keys：

MPT的实现​

key的类型​

以太坊中的Trie，key和value都是[]byte类型，如果要使用其他类型可以使用rlp进行编码转换为[]byte类型(rlp模块可以很方便的把各种结构序列化为[]byte)

nibble: key类型的最小单位， 内部实现里，将key中的每个字节按高4位和低4位拆分成了两个字节。比如你传入的key是：

在以太坊的实现中，索引数组的每个元素都是下一个结点的hash，而非指针。也就是说，父结点持有的是子结点的hash。

在以太坊的Trie中，遍历到叶子结点以后得到的是value的hash，而非value本身。使用这个hash从数据库中读取出来的值才是value。

当树的某条路径没有分支时，使用一个结点存储路径上的所有key

对于某个结点，如何知道它存储的是索引数组，还是key的数组呢？编码的时候在字节末尾会添加一个可选的截止符，用来表示是叶子节点还是拓展节点

以太坊在Trie内部将结点分成了两类：全节点（fullNode）和快捷节点（shortNode）。全节点有17个元素，而快捷结点有两个元素。它们的示意表达如下：

fullNode(branch node)： [i0, i2, … i15, value] shortNode(leaf node and extension node)： [ [k0, k1, … kn], value ]
